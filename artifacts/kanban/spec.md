## Overview
Trello 스타일의 칸반 Todo 앱. 3개 칼럼(To Do / In Progress / Done)에서 카드를 드래그&드롭으로 관리하고, 검색·필터·다크모드를 지원하며, 모든 데이터를 localStorage에 저장한다.

## 시나리오

### 1. 카드 추가
[상황] 사용자가 칼럼의 '추가' 버튼을 클릭하면
[동작] 제목 입력란이 나타나고, 제목을 입력 후 확인하면 해당 칼럼 하단에 새 카드가 추가된다

성공 기준:
- [ ] "To Do" 칼럼에서 '추가' 버튼 클릭 → 제목 입력란 표시
- [ ] 제목 "우유 사기" 입력 후 확인 → "To Do" 칼럼에 "우유 사기" 카드 표시
- [ ] 제목 비어있는 상태에서 확인 → "제목을 입력해주세요" 오류 메시지 표시

### 2. 카드 인라인 편집
[상황] 사용자가 카드를 클릭하면
[동작] 카드 상세 편집 화면이 열리고, 제목·설명·우선순위·태그·마감일을 수정할 수 있다

성공 기준:
- [ ] 카드 "우유 사기" 클릭 → 상세 편집 화면에 제목 "우유 사기" 표시
- [ ] 제목을 "빵 사기"로 변경 후 저장 → 카드 제목이 "빵 사기"로 표시
- [ ] 우선순위를 "높음"으로 변경 → 카드에 "높음" 우선순위 표시
- [ ] 태그 "장보기" 추가 → 카드에 "장보기" 태그 표시
- [ ] 마감일 "2026-03-25" 설정 → 카드에 마감일 표시

### 3. 카드 삭제
[상황] 사용자가 카드 상세 화면에서 '삭제' 버튼을 클릭하면
[동작] 확인 대화상자가 표시되고, 확인 시 카드가 삭제된다

성공 기준:
- [ ] '삭제' 버튼 클릭 → "정말 삭제하시겠습니까?" 확인 대화상자 표시
- [ ] 확인 클릭 → 카드가 목록에서 사라짐
- [ ] 취소 클릭 → 카드가 그대로 유지됨

### 4. 카드 드래그&드롭 이동
[상황] 사용자가 카드를 다른 칼럼으로 드래그하면
[동작] 카드가 원래 칼럼에서 사라지고 대상 칼럼에 표시된다

성공 기준:
- [ ] "To Do"의 카드를 "In Progress"로 드래그 → "To Do"에서 사라지고 "In Progress"에 표시
- [ ] "Done"의 카드를 "To Do"로 드래그 → 역방향 이동도 정상 동작
- [ ] 같은 칼럼 내 카드 순서 변경 → 드래그한 위치에 카드 배치

### 5. 서브태스크(체크리스트) 관리
[상황] 사용자가 카드 상세 화면에서 서브태스크를 추가하면
[동작] 체크리스트 항목이 추가되고, 체크/해제할 수 있다

성공 기준:
- [ ] 서브태스크 "계란 2판" 추가 → 체크리스트에 "계란 2판" 항목 표시
- [ ] "계란 2판" 체크 → 완료 표시 (취소선 등)
- [ ] 서브태스크 2개 중 1개 완료 → 카드에 "1/2" 진행률 표시

### 6. 제목 검색
[상황] 사용자가 검색란에 텍스트를 입력하면
[동작] 제목에 검색어가 포함된 카드만 실시간으로 필터링되어 표시된다

성공 기준:
- [ ] "우유" 입력 → "우유 사기" 카드만 표시, 다른 카드 숨김
- [ ] 검색어 지움 → 모든 카드 다시 표시
- [ ] 매칭 카드 없음 → "검색 결과가 없습니다" 안내 표시

### 7. 우선순위·태그 필터
[상황] 사용자가 필터 옵션에서 우선순위나 태그를 선택하면
[동작] 선택한 조건에 맞는 카드만 표시된다

성공 기준:
- [ ] 우선순위 "높음" 필터 선택 → 우선순위가 "높음"인 카드만 표시
- [ ] 태그 "장보기" 필터 선택 → "장보기" 태그가 있는 카드만 표시
- [ ] 우선순위 "높음" + 태그 "장보기" 동시 선택 → 두 조건 모두 만족하는 카드만 표시 (AND)
- [ ] 필터 해제 → 모든 카드 다시 표시

### 8. 다크모드 토글
[상황] 사용자가 다크모드 토글 버튼을 클릭하면
[동작] 전체 화면이 다크/라이트 테마로 전환된다

성공 기준:
- [ ] 라이트 모드에서 토글 클릭 → 다크 모드로 전환 (어두운 배경)
- [ ] 다크 모드에서 토글 클릭 → 라이트 모드로 전환 (밝은 배경)
- [ ] 페이지 새로고침 → 마지막 선택한 테마 유지

### 9. 데이터 영속성 (localStorage)
[상황] 사용자가 카드를 추가/수정/삭제/이동한 후 페이지를 새로고침하면
[동작] 모든 변경사항이 유지되어 이전 상태 그대로 표시된다

성공 기준:
- [ ] 카드 추가 후 새로고침 → 추가한 카드 그대로 표시
- [ ] 카드를 다른 칼럼으로 이동 후 새로고침 → 이동된 칼럼에 카드 표시
- [ ] 서브태스크 체크 후 새로고침 → 체크 상태 유지

### 10. 빈 칼럼 안내
[상황] 칼럼에 카드가 하나도 없으면
[동작] "카드가 없습니다" 안내 문구가 표시된다

성공 기준:
- [ ] 모든 카드를 다른 칼럼으로 이동 → "카드가 없습니다" 문구 표시
- [ ] 해당 칼럼에 카드 추가 → 안내 문구 사라짐

### 11. 마감일 초과 표시
[상황] 카드의 마감일이 오늘 날짜를 지났으면
[동작] 마감일이 초과 상태로 강조 표시된다

성공 기준:
- [ ] 마감일이 지난 카드 → 마감일이 초과(overdue) 상태로 표시
- [ ] 마감일이 오늘 이후인 카드 → 기본 상태로 표시

### 12. 검색과 필터 동시 적용
[상황] 사용자가 검색란에 텍스트를 입력하면서 필터도 선택하면
[동작] 검색어와 필터 조건을 모두 만족하는 카드만 표시된다

성공 기준:
- [ ] 검색어 "사기" + 우선순위 "높음" → "우유 사기(높음)", "빵 사기(높음)"만 표시
- [ ] 검색어 "우유" + 우선순위 "낮음" → 매칭 카드 없음, "검색 결과가 없습니다" 표시

## 범위

### 포함
- 3칼럼 칸반 보드 (To Do / In Progress / Done)
- 카드 CRUD (추가, 조회, 수정, 삭제)
- 카드 필드: 제목, 설명, 우선순위(높음/중간/낮음), 태그(복수, 자유 입력), 마감일
- 서브태스크 체크리스트 (추가, 체크/해제, 진행률)
- 드래그&드롭 (칼럼 간 이동, 칼럼 내 순서 변경)
- 제목 검색 (실시간 필터링)
- 우선순위·태그 필터 (AND 조건)
- 다크모드 토글
- localStorage 데이터 영속성

### 제외
- 사용자 인증/로그인: 단일 사용자 로컬 앱이므로 불필요
- 서버/API 연동: localStorage만 사용하는 클라이언트 앱
- 칼럼 추가/삭제/이름 변경: 3칼럼 고정으로 복잡도 제한
- 카드 첨부파일: localStorage 용량 제한으로 부적합
- 실시간 협업: 단일 사용자 앱

## 전제 조건
- React + TypeScript 기반 프로젝트
- shadcn/ui 컴포넌트 라이브러리 사용
- 드래그&드롭 라이브러리 사용 (예: @dnd-kit)
- 모던 브라우저 (localStorage 지원)
